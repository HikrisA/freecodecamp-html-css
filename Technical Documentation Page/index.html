Introduction

Microcontrollers are compact integrated circuits that serve as the brain of many embedded systems. They typically include a processor, memory, and input/output peripherals all on a single chip. Microcontrollers are widely used to automate simple tasks in electronic devices, from kitchen appliances to robots.

The PIC microcontroller family, developed by Microchip Technology, is one of the most popular and accessible microcontroller platforms in the world. The acronym "PIC" originally stood for Peripheral Interface Controller, though it now refers broadly to a diverse range of devices tailored for embedded applications.

PIC microcontrollers are well known for their simplicity, low cost, and wide availability. They are often used by students, hobbyists, and engineers alike due to their ease of programming, good documentation, and strong community support.

You can find PIC microcontrollers in a variety of real-world applications such as home automation systems, medical devices, automotive electronics, and industrial control systems.

Architecture

PIC microcontrollers are based on a modified Harvard architecture, which separates program memory from data memory. This allows the CPU to fetch instructions and data simultaneously, improving processing efficiency.

Most PIC microcontrollers include the following core components:

  • Central Processing Unit (CPU): Executes instructions and controls the operation of the microcontroller.
  • Flash Memory: Stores the program code. It is non-volatile, meaning it retains data even when powered off.
  • RAM (Random Access Memory): Used for temporary data storage during program execution.
  • EEPROM (Electrically Erasable Programmable Read-Only Memory): Stores data that must be preserved between resets or power cycles, such as configuration values.
  • I/O Ports (Input/Output): Allow the microcontroller to interact with external components such as LEDs, sensors, switches, and motors.
  • Timers and Counters: Used for precise time measurement and event counting.
  • Analog-to-Digital Converter (ADC): Converts analog signals from sensors into digital values for processing.

Depending on the specific model, a PIC microcontroller might also include additional features like PWM (Pulse Width Modulation) modules, serial communication interfaces (like UART, SPI, or I²C), and watchdog timers.

This modular and compact design makes PIC microcontrollers ideal for a wide variety of embedded applications.

PIC family

Microchip Technology has developed a wide range of PIC microcontrollers, each suited to different levels of complexity, performance, and cost. These families are generally grouped by their architecture and feature set. The most commonly used families include:

  • PIC10 Series: These are ultra-low-cost, small-footprint microcontrollers with very limited functionality. They are ideal for simple tasks such as controlling a single LED or button.
  • PIC12 Series: Slightly more advanced than PIC10, this series still offers compact designs but with more I/O pins and better features, such as integrated ADCs and timers.
  • PIC16 Series: A very popular family among students and hobbyists. These microcontrollers provide a good balance between performance and affordability, and support more advanced peripherals and programming options.
  • PIC18 Series: Designed for applications requiring higher performance, larger memory, and more advanced peripherals. They often support USB communication and complex timing functions.
  • PIC24 and dsPIC Series: These are 16-bit and 32-bit microcontrollers intended for high-performance and signal-processing applications. The dsPIC line combines a microcontroller with digital signal processing (DSP) capabilities.

Each family supports different instruction sets and development tools, but all share a consistent design philosophy that makes it easier to scale projects up or down based on performance needs.

Microcontrollers programmation

PIC microcontrollers can be programmed in several languages, including assembly and C. However, the most widely used language today is C, thanks to its readability, portability, and support through tools like MPLAB X IDE and the XC8 compiler, both provided by Microchip Technology.

A typical workflow when programming PICs includes:

  1. Writing the source code in C.
  2. Compiling it using the XC8 compiler.
  3. Uploading the generated .hex file to the microcontroller using a programmer such as a PICkit.

Below are some basic code snippets that illustrate common programming patterns used with PIC microcontrollers:

  1. Basic Program Structure 

    
#include 

void main(void) {
    // Initialization code
    while(1) {
        // Main loop
    }
}

  2. Setting a Pin as Output 

    
TRISB0 = 0; // Configure RB0 as an output
PORTBbits.RB0 = 0; // Set RB0 to logic low

  3. Delay Function 

                                
#define _XTAL_FREQ 4000000 // Define clock frequency
__delay_ms(500); // Delay for 500 milliseconds

  4. Toggling a Pin 

                                
PORTBbits.RB0 = !PORTBbits.RB0; // Toggle the state of RB0

  5. Configuration Bits (simplified) 

                                
#pragma config FOSC = INTRC_NOCLKOUT // Use internal oscillator
#pragma config WDTE = OFF            // Disable watchdog timer
#pragma config PWRTE = ON            // Enable power-up timer

These code snippets demonstrate how to configure and control the I/O ports of a PIC microcontroller using the XC8 compiler. More advanced programs can include interrupts, ADCs, PWM, and communication protocols like UART or I²C.

Example project

One of the most common beginner projects with a PIC microcontroller is making an LED blink. This simple task introduces you to working with I/O pins, configuring registers, and using delays—all essential skills for embedded development.

In this example, we will use a PIC microcontroller (such as the PIC16F877A) to turn an LED on and off with a 500 ms delay. The LED is connected to pin RB0.

Before we start, make sure you have:

  • MPLAB X IDE installed
  • The XC8 compiler
  • A programmer like PICkit 3 or 4
  • A breadboard and basic electronic components 
                    
#define _XTAL_FREQ 4000000
#include <xc.h>

// Configuration bits (example for PIC16F877A)
#pragma config FOSC = INTRC_NOCLKOUT
#pragma config WDTE = OFF
#pragma config PWRTE = ON
#pragma config BOREN = OFF
#pragma config LVP = OFF
#pragma config CPD = OFF
#pragma config WRT = OFF
#pragma config CP = OFF

void main(void) {
    TRISB0 = 0; // Set RB0 as output

    while(1) {
        PORTBbits.RB0 = 1; // Turn LED on
        __delay_ms(500);   // Wait 500 ms
        PORTBbits.RB0 = 0; // Turn LED off
        __delay_ms(500);   // Wait 500 ms
    }
}

How It Works

  • TRISB0 = 0; sets pin RB0 as an output.
  • PORTBbits.RB0 = 1; turns the LED on by sending 5V to the pin.
  • __delay_ms(500); waits 500 milliseconds.
  • Then it turns the LED off and repeats the cycle.

This is a simple yet powerful example that can be expanded into more complex applications like traffic lights, signal generators, or sensor-controlled systems.
